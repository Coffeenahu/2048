    <div class="tile tile-2" style="--r:0;--c:1">2</div>

**설계 포인트:**
- 타일은 `.cell`(배경 격자) 위에 절대 위치로 겹쳐 렌더링
- CSS 변수 `--r` (row), `--c` (col)로 위치 계산
- DOM 타일 재사용 없이 매 이동마다 전체 재렌더링 방식 사용 (단순성 우선)

### 모듈 구조 (단일 파일 내 함수 분리)

```
game.js
├── 상태 관리
│   ├── initGame()          초기화, 타일 2개 생성
│   └── saveState()         localStorage 베스트 스코어 저장

│   ├── render()            보드 전체 재렌더링

│   └── updateScore()       점수 UI 갱신

    ├── handleKey(e)        키보드 입력 처리
    ├── handleTouchStart(e) 터치 시작 좌표 저장
    └── handleTouchEnd(e)   스와이프 방향 계산 후 move() 호출
```

### 핵심 알고리즘: `slide(row)`

```js
// 입력: [2, 0, 2, 4]  →  출력: [4, 4, 0, 0], addedScore: 4
function slide(row) {
  // 1. 0 제거
  let tiles = row.filter(v => v !== 0);
  let addedScore = 0;

  // 2. 인접 동일값 병합 (왼쪽부터, 병합된 타일 재병합 불가)
  for (let i = 0; i < tiles.length - 1; i++) {
    if (tiles[i] === tiles[i + 1]) {
      tiles[i] *= 2;
      addedScore += tiles[i];
      tiles.splice(i + 1, 1);
    }
  }

  // 3. 우측 0 패딩
  while (tiles.length < 4) tiles.push(0);
  return { result: tiles, addedScore };
}
```

### 방향별 이동: `move(dir)`

```js
function move(dir) {
  let moved = false;

  for (let i = 0; i < 4; i++) {
    // 방향에 따라 행/열 추출 → slide() → 결과 반영
    // 'left':  board[i]
    // 'right': board[i].slice().reverse() → slide → reverse
    // 'up':    board의 i번째 열 추출
    // 'down':  board의 i번째 열 reverse
  }

  if (moved) {
    addTile();
    render();
    if (isGameOver()) showOverlay();
    saveState();
  }
}
```

### 터치 처리

```js
let touchStartX, touchStartY;
const SWIPE_THRESHOLD = 30; // px

board.addEventListener('touchstart', e => {
  touchStartX = e.touches[0].clientX;
  touchStartY = e.touches[0].clientY;
});

board.addEventListener('touchend', e => {
  const dx = e.changedTouches[0].clientX - touchStartX;
  const dy = e.changedTouches[0].clientY - touchStartY;

  if (Math.abs(dx) < SWIPE_THRESHOLD && Math.abs(dy) < SWIPE_THRESHOLD) return;

  if (Math.abs(dx) > Math.abs(dy)) {
    move(dx > 0 ? 'right' : 'left');
  } else {
    move(dy > 0 ? 'down' : 'up');
  }
});
```

## 6. 설계 결정 사항

| 결정 | 선택 | 대안 | 이유 |
|------|------|------|------|
| 렌더링 방식 | 전체 재렌더링 | DOM diff | 단순성 우선, 4×4라 성능 문제 없음 |
| 타일 위치 | CSS 변수 (`--r`,`--c`) | transform translate | 코드 직관적, 전환 애니메이션 자연스러움 |
| 상태 저장 | localStorage (베스트만) | 전체 게임 상태 저장 | 복잡도 최소화, 재시작이 주 UX |
| 파일 분리 | 단일 3파일 | 모듈 분리 | 빌드 도구 없는 환경에 적합 |
